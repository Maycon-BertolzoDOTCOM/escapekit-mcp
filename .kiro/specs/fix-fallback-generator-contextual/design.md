# Design Document

## Overview

Refatoração do `FallbackGenerator` para substituir o código hardcoded por templates Handlebars por biblioteca WebGL, adicionar detecção do framework UI do projeto, e implementar integração automática do fallback ao entry point. A mudança é aditiva: dois novos métodos privados são adicionados (`detectUIFramework` e `integrateIntoEntryPoint`), o método `generateFallbackCode` é refatorado para usar templates, e o método `fix` é estendido para chamar a integração ao entry point após criar o Fallback_File. A interface pública `Fixer` e o `AutoFixEngine` não requerem nenhuma modificação.

---

## Architecture

### Componentes Envolvidos

```
AutoFixEngine
    └── FallbackGenerator (refatorado)
            ├── detectWebGLLibrary(projectPath)   → 'three' | 'babylon' | 'pixi' | 'r3f' | null
            ├── detectUIFramework(projectPath)     → 'react' | 'vue' | 'svelte' | 'vanilla'  [NOVO]
            ├── generateFallbackCode(library)      → string  [refatorado: usa templates]
            │       └── renderTemplate(library)    → string  [NOVO]
            │               └── Handlebars.compile(templateSource)(context)
            └── integrateIntoEntryPoint(projectPath, uiFramework)  [NOVO]
                    └── prepend import ao Entry_Point encontrado

templates/fallback/
    ├── three.hbs    [NOVO]
    ├── babylon.hbs  [NOVO]
    ├── pixi.hbs     [NOVO]
    ├── r3f.hbs      [NOVO]
    └── generic.hbs  [NOVO]
```

### Fluxo do método `fix` estendido

```
fix(projectPath, issue)
    │
    ├─ 1. detectWebGLLibrary(projectPath)   → webglLib  (existente)
    ├─ 2. detectUIFramework(projectPath)    → uiFramework  [NOVO]
    ├─ 3. mkdir src/utils/ se não existir   (existente)
    ├─ 4. generateFallbackCode(webglLib)    → código via template ou hardcoded
    │       └── renderTemplate(webglLib)    → Handlebars ou graceful degradation
    ├─ 5. writeFile src/utils/webgl-fallback.ts  (existente)
    ├─ 6. mkdir src/styles/ se não existir  (existente)
    ├─ 7. writeFile src/styles/webgl-fallback.css  (existente)
    └─ 8. integrateIntoEntryPoint(projectPath, uiFramework)  [NOVO]
            └── retorna { file?: string; note?: string }
```

## Detailed Design

### 1. Templates Handlebars

Cada template exporta as funções `checkWebGLSupport` e `setupFallback` para manter a interface consistente entre bibliotecas. As variáveis disponíveis são: `{{containerSelector}}`, `{{width}}`, `{{height}}`, `{{projectName}}`.

```tsx
export function WebGLFallback({ width = {{width}}, height = {{height}} }: { width?: number; height?: number }) {
  const canvasRef = useRef<HTMLCanvasElement>(null);

  useEffect(() => {
    const canvas = canvasRef.current;
    if (!canvas) return;
    const ctx = canvas.getContext('2d');

    <div className="webgl-fallback" data-fallback="true">
      <canvas ref={canvasRef} width={width} height={height} style={{ width: '100%', height: '100%' }} />
    </div>
  );
}
```

### 2. Método `detectUIFramework`

```typescript
private async detectUIFramework(projectPath: string): Promise<UIFramework> {
  try {
    const pkgPath = join(projectPath, 'package.json');
    const pkg = JSON.parse(await readFile(pkgPath, 'utf-8'));
    const allDeps = { ...pkg.dependencies, ...pkg.devDependencies };

    if (allDeps['react']) return 'react';
    if (allDeps['vue']) return 'vue';
    if (allDeps['svelte']) return 'svelte';
    return 'vanilla';
  } catch {
    return 'vanilla';
  }
}
```


### 3. Método `renderTemplate`

Carrega o template do disco e renderiza com Handlebars. Em caso de falha (módulo ausente ou arquivo não encontrado), retorna `null` para acionar o graceful degradation.

```typescript
private async renderTemplate(library: string | null, context: TemplateContext): Promise<string | null> {
  try {
    const Handlebars = await import('handlebars');
    const templateName = library ?? 'generic';
    const templatePath = join(
      new URL('../../..', import.meta.url).pathname,
      'templates', 'fallback', `${templateName}.hbs`
    );
    const source = await readFile(templatePath, 'utf-8');
    const compiled = Handlebars.default.compile(source);
    return compiled(context);
  } catch {
    return null;
  }
}
```

### 4. Método `generateFallbackCode` refatorado

```typescript
private async generateFallbackCode(library: string | null, context: TemplateContext): Promise<string> {
  const rendered = await this.renderTemplate(library, context);
  if (rendered !== null) return rendered;

  // Graceful degradation: usa código hardcoded atual
  this.log.warn('Handlebars templates not available, using hardcoded fallback', { library });
  return this.generateHardcodedFallback(library);
}
```

O método `generateHardcodedFallback` é o código atual de `generateFallbackCode` renomeado, preservando o comportamento existente integralmente.

**Nota:** A assinatura de `generateFallbackCode` muda de `(library: string | null): string` para `(library: string | null, context: TemplateContext): Promise<string>`. O método `fix` é atualizado para usar `await`.

### 5. Método `integrateIntoEntryPoint`

```typescript
private async integrateIntoEntryPoint(
  projectPath: string,
  uiFramework: UIFramework
): Promise<{ file?: string; note?: string }> {
  const IMPORT_LINE = "import { checkWebGLSupport, setupFallback } from './utils/webgl-fallback';";
  const IMPORT_MARKER = "from './utils/webgl-fallback'";

  const candidatesByFramework: Record<UIFramework, string[]> = {
    react:   ['src/App.tsx', 'src/App.jsx', 'src/main.tsx', 'src/main.jsx'],
    vue:     ['src/App.vue', 'src/main.ts', 'src/main.js'],
    svelte:  ['src/App.svelte', 'src/main.ts'],
    vanilla: ['src/main.ts', 'src/index.ts', 'src/main.js', 'src/index.js'],
  };

  const candidates = candidatesByFramework[uiFramework];

  for (const candidate of candidates) {
    const fullPath = join(projectPath, candidate);
    try {
      await access(fullPath);
      const content = await readFile(fullPath, 'utf-8');
      if (content.includes(IMPORT_MARKER)) {
        return { file: candidate }; // já existe, idempotente
      }
      await writeFile(fullPath, `${IMPORT_LINE}\n${content}`, 'utf-8');
      this.log.info('Integrated fallback import into entry point', { file: candidate });
      return { file: candidate };
    } catch {
      continue;
    }
  }

  this.log.warn('Entry point not found for fallback integration', { uiFramework, candidates });
  return { note: `Entry point not found for ${uiFramework} — add import manually` };
}
```

### 6. Método `fix` estendido

```typescript
async fix(projectPath: string, issue: Issue): Promise<Fix> {
  this.log.info('Generating WebGL fallback', { message: issue.message });

  const webglLib = await this.detectWebGLLibrary(projectPath);
  const uiFramework = await this.detectUIFramework(projectPath);  // NOVO

  const fallbackPath = join(projectPath, 'src', 'utils', 'webgl-fallback.ts');
  const fallbackDir = join(projectPath, 'src', 'utils');

  try {
    try {
      await access(fallbackDir);
    } catch {
      await mkdir(fallbackDir, { recursive: true });
    }

    // Ler projectName para o TemplateContext
    let projectName = 'project';
    try {
      const pkg = JSON.parse(await readFile(join(projectPath, 'package.json'), 'utf-8'));
      if (pkg.name) projectName = pkg.name;
    } catch { /* usa padrão */ }

    const context: TemplateContext = {
      containerSelector: '#app',
      width: 800,
      height: 600,
      projectName,
    };

    const fallbackCode = await this.generateFallbackCode(webglLib, context);  // agora async
    await writeFile(fallbackPath, fallbackCode, 'utf-8');
    this.log.info('Created WebGL fallback utility', { path: fallbackPath });

    // CSS (existente)
    const cssPath = join(projectPath, 'src', 'styles', 'webgl-fallback.css');
    const cssDir = join(projectPath, 'src', 'styles');
    try { await access(cssDir); } catch { await mkdir(cssDir, { recursive: true }); }
    await writeFile(cssPath, this.generateFallbackCSS(), 'utf-8');

    // Integração ao entry point  [NOVO]
    const integration = await this.integrateIntoEntryPoint(projectPath, uiFramework);

    const usedTemplate = webglLib ?? 'generic';
    const description = integration.note
      ? `Generated WebGL fallback for ${usedTemplate} using template — ${integration.note}`
      : `Generated WebGL fallback for ${usedTemplate} using template`;

    return {
      issueType: issue.type,
      description,
      file: integration.file ?? 'src/utils/webgl-fallback.ts',
      applied: true,
    };
  } catch (err) {
    return {
      issueType: issue.type,
      description: `Failed to generate fallback: ${err instanceof Error ? err.message : String(err)}`,
      applied: false,
      error: err instanceof Error ? err.message : String(err),
    };
  }
}
```

## Correctness Properties

### P1 — Three.js: arquivo gerado contém CSS2DRenderer ou WebGLRenderer

Para qualquer projeto com `three` em `dependencies` ou `devDependencies`, o Fallback_File gerado DEVE conter `CSS2DRenderer` ou `WebGLRenderer`.

Formalmente: `webglLib === 'three' → fallbackCode.includes('CSS2DRenderer') || fallbackCode.includes('WebGLRenderer')`

### P2 — R3F: arquivo gerado contém JSX válido

Para qualquer projeto com `@react-three/fiber` em `dependencies` ou `devDependencies`, o Fallback_File gerado DEVE conter marcadores de JSX (`<` e `/>`).

Formalmente: `webglLib === 'r3f' → fallbackCode.includes('<') && fallbackCode.includes('/>')`

### P3 — Idempotência da integração ao entry point

Aplicar `integrateIntoEntryPoint` duas vezes consecutivas no mesmo projeto DEVE produzir o mesmo estado final que aplicar uma única vez — o import não é duplicado.

Formalmente: `f(f(x)).entryPointContent === f(x).entryPointContent`

### P4 — Fallback_File sempre criado para qualquer biblioteca

Para qualquer WebGL_Library detectada (incluindo `null`), o arquivo `src/utils/webgl-fallback.ts` DEVE ser criado no projeto alvo quando `fix` retornar `applied: true`.

Formalmente: `result.applied === true → fs.existsSync(join(projectPath, 'src/utils/webgl-fallback.ts'))`

### P5 — Graceful degradation: sem Handlebars não lança erro

Quando Handlebars não estiver disponível, `fix` DEVE retornar `{ applied: true }` com código genérico — nunca `applied: false` por ausência de Handlebars.

Formalmente: `!handlebarsAvailable → result.applied === true && result.error === undefined`

## Error Handling

| Situação | Comportamento |
|---|---|
| `handlebars` não instalado | Log `warn`, usa código hardcoded, continua normalmente |
| Arquivo `.hbs` não encontrado | Log `warn`, usa código hardcoded, continua normalmente |
| `package.json` não encontrado | `projectName = 'project'`, `webglLib = null`, `uiFramework = 'vanilla'` |
| Entry point não encontrado | Log `warn`, retorna `Fix { applied: true }` com nota no `description` |
| Import já presente no entry point | Não modifica o arquivo, retorna `Fix { applied: true }` |
| Falha de I/O ao criar `src/utils/` | Retorna `Fix { applied: false, error: <mensagem> }` |
| Falha de I/O ao escrever Fallback_File | Retorna `Fix { applied: false, error: <mensagem> }` |
| Falha de I/O ao escrever CSS | Retorna `Fix { applied: false, error: <mensagem> }` |

## Implementation Notes

- O caminho dos templates é resolvido relativamente ao arquivo `FallbackGenerator.ts` usando `import.meta.url` para compatibilidade com ESM. O caminho resultante é `../../templates/fallback/{library}.hbs` a partir de `src/validate/auto-fix/`.
- O import dinâmico `await import('handlebars')` é usado para permitir graceful degradation sem `try/catch` no nível do módulo.
- O método `generateFallbackCode` muda de síncrono para assíncrono (`Promise<string>`). O método `fix` já é `async`, então o `await` é adicionado sem impacto na interface pública.
- O método `generateHardcodedFallback` é o código atual de `generateFallbackCode` renomeado. Isso preserva o comportamento existente como fallback de último recurso.
- O `AutoFixEngine` não requer nenhuma modificação — `new FallbackGenerator()` continua funcionando sem argumentos.
- A dependência `handlebars` deve ser adicionada ao `package.json` do EscapeKit MCP (não do projeto alvo). Se já estiver presente, nenhuma ação é necessária.
- Os templates usam `{{width}}` e `{{height}}` como números literais (sem aspas) para que o código gerado seja TypeScript válido.
