# 📜 Manifesto Ralph Loop Inverso

> *"Code that cannot be migrated is not owned by the user."*

---

## The Problem We Solve

AI platforms generate code that works beautifully inside their sandboxes — and breaks catastrophically outside them. Ghost dependencies, mock APIs, ephemeral polyfills. The developer becomes a **captive resource** maintaining someone else's ecosystem.

We call this the **Ralph Loop Inverso**: a dependency cycle where the creator becomes the prisoner of the creation environment.

---

## Our Principles

### 1. 🔓 Code Portability Is a Right, Not a Feature

Every line of code generated by AI belongs to the developer. If that code only runs inside a specific sandbox, the developer doesn't truly own it. EscapeKit exists to break these invisible chains.

### 2. 🛡️ Security Is Non-Negotiable

Ghost dependencies are not just a convenience problem — they're a **supply-chain attack vector**. A hallucinated package name can be maliciously registered on npm (typosquatting). Detection of security threats is always **Priority Zero**.

### 3. 🧠 AST Over Regex, Always

Code analysis must be precise. We use Tree-sitter for Abstract Syntax Tree parsing — never regex for structural code patterns. Precision beats cleverness; correctness beats speed.

### 4. 🌊 Graceful Degradation

Network failures, malformed code, missing packages — the real world is messy. EscapeKit must never crash ungracefully. Every failure path produces a useful warning, never a silent error or an unhandled exception.

### 5. 🏗️ Simple > Clever

We follow KISS, DRY, and SOLID. Code should be readable by a developer who's never seen the project. If a solution requires a comment to explain *why it's clever*, it's probably too clever.

### 6. 🌍 Developer Autonomy

The tool serves the developer, not the other way around. No mandatory telemetry. No vendor lock-in. Community edition is always free and fully functional for individual developers.

### 7. 🔬 Evidence-Based Evolution

Every feature should be measurable. We track activation rate (how many analyses lead to successful migrations), not vanity metrics. Improvement triggers are driven by real-world signals, not roadmap pressure.

---

## The Vision

A world where AI code generation is truly **generative** — creating assets the developer fully controls, deploys anywhere, and evolves independently. The sandbox is a starting point, never a prison.

**EscapeKit: Breaking Ralph Loop Inverso, one sandbox at a time.** 🚀
